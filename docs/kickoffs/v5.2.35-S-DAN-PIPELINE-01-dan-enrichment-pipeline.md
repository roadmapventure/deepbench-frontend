## 2. CONTEXT

### What this session fixes

BUG-17 has three confirmed gaps preventing Dan Bingham (PS-01, AI Prompt Strategist) from contributing his authored prompts to the enrichment pipeline:

**Gap 1 — Legacy `technical_services` on Michelle's `project-manager` capability (Supabase)**

Two skill profiles in the `project-manager` capability still carry enrichment triggers from before Dan was created (SK-16, S-PM-02):
- `planning-behavior`: `technical_services: ["reflect"]` — triggers reflect but Dan's `traits.reflect_prompt` is never reached; generic fallback runs instead
- `work-order-decomposition`: `technical_services: ["intelligent-synthesis"]` — triggers synthesis but Dan's `traits.synthesis_prompt` is never reached; generic fallback runs instead

Michelle's PM capability should be purely declarative. She declares sections (behavior, intent, knowledge, format). She does not trigger LLM enrichment calls.

**Gap 2 — `dan-ai-enrichment` capability never assembled by `plan.js`**

Dan's skill profiles are correctly configured in Supabase (confirmed):
- `prompt-reflect`: `skill_type_slug: 'behavior'`, `technical_services: ["reflect"]`, `traits.reflect_prompt` = Dan's authored reflect instruction
- `prompt-synthesis`: `skill_type_slug: 'intent'`, `technical_services: ["intelligent-synthesis"]`, `traits.synthesis_prompt` = Dan's authored synthesis instruction

But `plan.js` only calls `assemblePrompt({ capability_slug: 'project-manager', ... })`. Dan's `dan-ai-enrichment` capability is never loaded. His prompts are dead code.

**Gap 3 — `ai-enrichment` SERVICE_CATALOG.patterns missing `'Reflection'`**

`useAIActivity.js` SERVICE_CATALOG entry for `ai-enrichment` lists `['RAG', 'Prompt Chaining']` — missing `'Reflection'`, which is `active: true` in PATTERN_CATALOG (fixed in BUG-15).

### Architecture principles locked this session

- **Michelle's PM capability is declarative only.** No `technical_services` enrichment triggers on any of her skill profiles after this session.
- **Dan owns the enrichment layer.** All reflect and synthesis triggers come exclusively from `dan-ai-enrichment` capability skill profiles.
- **`assemblePrompt()` gains `enrichment_capability_slug` param.** When provided, the enrichment capability's skill profiles are loaded alongside the primary capability and merged into the same `buildSections()` call. Dan's `prompt-reflect` and `prompt-synthesis` have null section content (no visible text output) but carry `technical_services` triggers. This is the correct behavior.
- **No hard-coded service calls.** Dan's enrichment is assembled as content that enriches the prompt before it reaches the LLM. One pipeline: assemble → enrich → send. No new call sites.

### Why Dan's null-content skill profiles don't pollute the prompt

`prompt-reflect` is `skill_type_slug: 'behavior'`. In `buildSections()`, a behavior section's content is assembled from `agentConfigs` (role_prompts) and `traits.reasoning_style` / `traits.writing_style`. Dan's `dan-ai-enrichment` capability has no `agent_configs` for the active agent (Michelle), and `prompt-reflect` has none of those traits. So `content = null`. In `ai-enrichment.js`, null-content sections are skipped at render (`if (!s.content) { omitted.push(s.slug); continue; }`). The section lands in the sections array, carries the `technical_services` trigger, but produces no visible prompt text.

Same logic applies to `prompt-synthesis` (`skill_type_slug: 'intent'` — requires `objective`, `method`, or `traits.analysis_instructions`, which are not set on this skill profile).
