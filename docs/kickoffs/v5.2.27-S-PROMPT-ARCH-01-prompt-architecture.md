## Section 2 — CONTEXT

### What this session does

Four connected pipeline improvements to the Prompt Service:

**AA-60 — REFLECT prompt from traits**
Dan Bingham's `prompt-reflect` skill profile has `technical_services: ["reflect"]`. Currently DB Assembly builds the reflectSection with a hardcoded `fetch_instruction` — there is no field for a custom prompt. AI Enrichment uses a hardcoded REFLECT prompt string. Fix: DB Assembly passes `traits.reflect_prompt` in the reflectSection `fetch_instruction`. AI Enrichment reads `fi.reflect_prompt` and uses it when present; falls back to the existing hardcoded prompt when absent.

**AA-61 — Synthesis prompt from traits**
Dan Bingham's `prompt-synthesis` skill profile has `technical_services: ["intelligent-synthesis"]`. Currently the Synthesis step in AI Enrichment uses a hardcoded instruction. Fix: DB Assembly passes `traits.synthesis_prompt` in the `synthesis` object. AI Enrichment reads `synthesis.prompt` and uses it when present; falls back to hardcoded when absent. The token limit instruction (`The rewritten prompt must be under N tokens`) and the assembled prompt are always appended in code after the trait instruction — they are never stored in the trait.

**AA-62 + AA-67 — WORK ORDER section includes deliverable_type**
The existing WORK ORDER section in DB Assembly (AA-57) injects only `task_context.goal`. `task_context.deliverable_type` is silently dropped and never reaches the LLM. Fix: extend the WORK ORDER section content to include `deliverable_type` when present. The section stays at order 2.5, type `stored`, always injected when goal exists — visible in all 4 columns of the Prompt Evolution Modal. No architectural change to where the section lives.

### Why this matters
After this session, every prompt the Prompt Service sends to Anthropic is fully data-driven. Dan Bingham's REFLECT and Synthesis steps run on his authored instructions, not hardcoded strings. The LLM receives both the goal and the deliverable type in the WORK ORDER section. To tune Dan's reasoning quality: update a trait in Supabase. No deploy required.

function buildSections_mock(skillProfiles) {
  let reflectSection = null;
  let synthesisEnabled = false;
  let synthesisDeclaringSlug = null;
  let synthesisPromptText = null;

  for (const sp of skillProfiles) {
    const traits = sp.traits || {};
    const techServices = Array.isArray(sp.technical_services) ? sp.technical_services : [];

    if (techServices.includes("reflect") && !reflectSection) {
      reflectSection = {
        slug: "reflect",
        type: "reflect",
        fetch_instruction: {
          method: "reflect",
          model: "claude-haiku-4-5-20251001",
          max_tokens: 1024,
          inserts_after: "behavior",
          declared_by: sp.slug,
          reflect_prompt: traits.reflect_prompt || null,
        },
      };
    }

    if (techServices.includes("intelligent-synthesis") && !synthesisEnabled) {
      synthesisEnabled = true;
      synthesisDeclaringSlug = sp.slug;
      synthesisPromptText = traits.synthesis_prompt || null;
    }
  }

  const synthesis = synthesisEnabled
    ? { enabled: true, model: "claude-haiku-4-5-20251001", max_tokens: 2048, declared_by: synthesisDeclaringSlug, prompt: synthesisPromptText }
    : { enabled: false };

  return { reflectSection, synthesis };
}

function buildWorkOrderContent(task_context) {
  const goalText = typeof task_context === 'object' && task_context !== null
    ? (task_context.goal || null)
    : (typeof task_context === 'string' ? task_context : null);
  const deliverableType = typeof task_context === 'object' && task_context !== null
    ? (task_context.deliverable_type || null)
    : null;

  if (!goalText || !goalText.trim()) return null;

  const parts = [`Goal: ${goalText.trim()}`];
  if (deliverableType) parts.push(`Deliverable type: ${deliverableType}`);
  return parts.join('\n');
}
