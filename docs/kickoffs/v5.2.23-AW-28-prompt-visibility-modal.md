## Section 2 — CONTEXT

When a user clicks "Generate Plan" on the Create Work Order screen, two things happen in parallel:

1. `action: 'prompt-service'` — the full pipeline runs (DB Assembly → AI Enrichment → Request & Receivable). This produces the actual plan.
2. `action: 'preview-prompt'` — a new action on `plan.js` that runs DB Assembly + AI Enrichment only (no Request & Receivable), returns the assembled prompt data in three stages.

When `preview-prompt` resolves, a modal opens immediately — **before** the plan is ready — showing the user how DeepBench built the prompt. This is a product differentiation moment: the user sees in real time that their prompt went from a 1-sentence goal to a richly assembled system prompt, and can see exactly which platform capabilities contributed.

**The modal shows three columns simultaneously:**

**Column 1 — "Generic AI Prompt"**  
Label: "What most AI platforms see"  
Content: The raw goal text only (~18 tokens). Intentionally sparse — the contrast is the story.

**Column 2 — "Data-Connected Prompt"**  
Label: "What a platform with a database adds"  
Content: The DB Assembly output — stored sections (Identity, Intent, Behavior, Format) rendered as labeled blocks. No RAG, no Reflect yet.

**Column 3 — "DeepBench Enriched"**  
Label: "What DeepBench sends to the AI"  
Content: The full enriched system prompt. RAG knowledge and Reflect execution plan sections are present, marked with inline `[RAG]` and `[REFLECT]` badges.

Token count shown under each column. Delta shown between columns ("+292 tokens", "+580 tokens"). Pattern badges shown for what fired (RAG ✓, Reflect ✓/✗, Synthesis ✓/✗).

Bottom bar: spinner ("Running plan generation...") while `prompt-service` is in flight; text changes to "Plan ready" when done. Continue button always enabled — closes modal and processes plan result.

```js
// FEATURE: AW-28 — preview-prompt: returns 3-stage prompt comparison for Prompt Evolution Modal
if (action === 'preview-prompt') {
  try {
    const { agent_id, capability_slug, tenant_id = 'global', goal, deliverable_type, runtime_context } = req.body;

    if (!goal) return res.status(400).json({ error: 'goal required' });

    // Stage 1: raw goal only
    const stage1Text = goal.trim();
    const stage1Tokens = Math.ceil(stage1Text.length / 4);

    // Stage 2: DB Assembly (stored sections only — no RAG, no Reflect)
    const promptRequest = await assemblePrompt({
      capability_slug: capability_slug || 'project-manager',
      agent_id: agent_id || 'michelle',
      tenant_id,
      task_context: { goal, deliverable_type },
      runtime_context: runtime_context || null,
    });

    const storedSections = (promptRequest.sections || []).filter(s => s.type === 'stored' && s.content);
    const stage2Text = storedSections
      .map(s => `=== ${s.label} ===\n${s.content}`)
      .join('\n\n---\n\n');
    const stage2Tokens = Math.ceil(stage2Text.length / 4);
    const stage2Sections = storedSections.map(s => ({ slug: s.slug, label: s.label, content: s.content, source: 'DB' }));

    // Stage 3: AI Enrichment (full assembled prompt)
    const enriched = await enrichPrompt({ prompt_request: promptRequest, agent_id, capability_slug });

    const stage3Text = enriched.system_prompt || '';
    const stage3Tokens = Math.ceil(stage3Text.length / 4);

    // Build stage 3 sections list with source badges
    const stage3Sections = Object.entries(enriched.sections || {}).map(([slug, content]) => {
      let source = 'DB';
      if (slug.startsWith('knowledge')) source = 'RAG';
      else if (slug === 'reflect') source = 'REFLECT';
      else if (slug === 'work-order') source = 'DB';
      return { slug, label: slug.replace(/-/g, ' ').toUpperCase(), content, source };
    });

    // Patterns that fired
    const debug = enriched.debug || {};
    const patterns = {
      rag: debug.rag_retrieved || false,
      rag_chunks: debug.rag_chunks_by_section ? Object.values(debug.rag_chunks_by_section).reduce((a, b) => a + b, 0) : 0,
      rag_scope: debug.rag_scope_effective || null,
      reflect: debug.reflect_ran || false,
      synthesis: debug.synthesis_ran || false,
      model: enriched.llm?.model || 'claude-sonnet-4-6',
      skill_profile: enriched.format_contract?.skill_profile_slug || null,
    };

    return res.status(200).json({
      stage1: { text: stage1Text, tokens: stage1Tokens },
      stage2: { text: stage2Text, tokens: stage2Tokens, sections: stage2Sections },
      stage3: { text: stage3Text, tokens: stage3Tokens, sections: stage3Sections },
      patterns,
    });
  } catch (e) {
    console.error('[preview-prompt] error:', e);
    return res.status(500).json({ error: e.message });
  }
}
```
