## 2. CONTEXT

**Root cause of QA FAIL item 11:**

`RosterScreen.jsx` line 39 calls `agent.trainableBy.toUpperCase()` on every agent in the AGENTS array. Victoria's entry was written with only unique/custom fields (`isProductStrategist`) and omitted all standard agent fields. `trainableBy` is `undefined` → `undefined.toUpperCase()` crashes the Bench tab entirely.

Additional missing fields that would break other renders: `quip` (line 52), `skill` (line 59/61), `color` (line 59/61/133+), `trainable` (line 19/20/132), `docs`, `classes`, `chunks`, `situational`, `revenueModel`.

**The fix:** Complete Victoria's agents.js entry with all standard fields. Once complete, she automatically appears on the Bench Roster and gets a Personnel File page — no other code changes needed. The components already handle any agent in the AGENTS array.

**New standard (locked this session):** Every agent build must ship a complete agents.js entry. No partial entries. See STANDARDS.md Section 11 (added this session).

## Section 11: Agent Build Completeness Standard

| `value` | number | Billed value |
| `hourly` | number | Hourly rate |
| `reportHrs` | number | Avg hours per report |
| `reportCost` | number | hourly × reportHrs |
| `docs` | number | Training docs count |
| `classes` | number | Training classes count |
| `chunks` | number | RAG chunks count |
| `skill` | number | Skill score 0–100 |
