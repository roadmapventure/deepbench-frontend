## SECTION 2 — CONTEXT

`request-receivable.js` is the third and final route in the Prompt Service pipeline:

```
DB Assembly  →  AI Enrichment  →  Request & Receivable
(assemble)      (enrich prompt)    (send + parse + guard + store)
```

This route:
1. **Receives** the assembled + enriched prompt (sections array, format_contract, llm config) plus `task_id` and `agent_id`
2. **Sends** to Anthropic (direct fetch, no adapter). Uses format_contract.output_type to choose call shape: `json` → tool_choice structured output using format_contract.schema; `html` or other → free-text
3. **Parses** the response. On schema mismatch for JSON output: retry once with corrective message appended. After second failure: return error
4. **Guardrails** (PAT-13): if `format_contract.guardrails.must` or `must_not` is non-empty, POST to Haiku with the parsed response + guardrails list and ask for pass/fail check. Log as a separate `ai_activity_log` row with `patterns_used: ['guardrails']`
5. **Delivers** to handler: delegates to `api/lib/handlers/[format_contract.handler].js`. Only `store.js` ships in this session
6. **Logs** entire operation server-side to `ai_activity_log` via Supabase service key (NOT via `logAICall()` frontend hook)
7. **Returns** structured output to caller: `{ deliverable_id, title, handler, guardrails_passed, violations, patterns_used, debug }`

**What this does NOT include:**
- No frontend wiring to this route — that is S-PM-05 (design session first)
- No `dispatch.js`, `package.js`, or `mcp.js` handlers — those are future (AA-54)
- No streaming in this session — the current call shape is full-response JSON or HTML

// api/lib/handlers/store.js
export async function handle({ task_id, agent_id, skill_profile_slug, title, content, format, handler, supabase }) {
  const { data, error } = await supabase
    .from('deliverables')
    .insert({ tenant_id: 'global', task_id, agent_id, skill_profile_slug, title, content, format, handler, status: 'draft', is_final: false })

    .single();
  if (error) throw error;
  return { deliverable_id: data.id };

## SECTION 6 — DETAILED SPEC

### Step 1 — Send to LLM

```js
const isJson = format_contract.output_type === 'json';
const model = llm?.model || 'claude-sonnet-4-6';
const max_tokens = llm?.max_tokens || 2048;

const systemPrompt = sections.map(s => `=== ${s.label} ===\n${s.content}`).join('\n\n');

let callBody;
if (isJson && format_contract.schema) {
  callBody = {
    model,
    max_tokens,
    tools: [{ name: format_contract.skill_profile_slug, description: 'Return structured output', input_schema: format_contract.schema }],
    tool_choice: { type: 'tool', name: format_contract.skill_profile_slug },
    messages: [{ role: 'user', content: systemPrompt }],
  };
} else {
  callBody = {
    model,
    max_tokens,
    system: systemPrompt,
    messages: [{ role: 'user', content: 'Please complete the task as instructed.' }],
  };
}
```

Parse JSON response from tool_use block. Parse text response from content[0].text. Extract `title` from parsed output (present because AI Enrichment Format section appends title instruction).

**Retry logic (JSON only):** if parse fails or required schema fields missing, append corrective user message and retry once. On second failure, return 422 error with debug info.

### Step 2 — Guardrails (PAT-13)

Only runs when `format_contract.guardrails.must.length > 0` OR `format_contract.guardrails.must_not.length > 0`.

```js
const guardrailsModel = 'claude-haiku-4-5-20251001';
const guardrailsPrompt = `You are a content validator. Review the following AI output and check it against the rules below.

OUTPUT:
${JSON.stringify(parsedResponse)}

MUST contain:
${format_contract.guardrails.must.map(r => `- ${r}`).join('\n') || '(none)'}

MUST NOT contain:
${format_contract.guardrails.must_not.map(r => `- ${r}`).join('\n') || '(none)'}

Return JSON: { "passed": true|false, "violations": ["list of rule violations, or empty"] }`;

const guardrailsResult = await fetch('https://api.anthropic.com/v1/messages', {
  method: 'POST',
  headers: { 'x-api-key': process.env.ANTHROPIC_API_KEY, 'anthropic-version': '2023-06-01', 'content-type': 'application/json' },
  body: JSON.stringify({
    model: guardrailsModel,
    max_tokens: 256,
    tools: [{ name: 'guardrails_check', description: 'Validate output', input_schema: { type: 'object', properties: { passed: { type: 'boolean' }, violations: { type: 'array', items: { type: 'string' } } }, required: ['passed', 'violations'] } }],
    tool_choice: { type: 'tool', name: 'guardrails_check' },
    messages: [{ role: 'user', content: guardrailsPrompt }],
  }),
});
```

Log guardrails call as a second `ai_activity_log` row with `type: 'guardrails-check'`, `patterns_used: ['guardrails', 'prompt-chaining']`.

If `passed: false`: set `guardrails_passed = false`, populate `violations`. Do NOT block delivery — log and pass through. (Blocking on guardrail failure is a future enhancement — AA-54 design scope.)

Write directly to Supabase using service key (not `logAICall()` frontend hook). Write happens AFTER handler completes.
