## 2. CONTEXT

`AgentAvatar` in `SharedUI.jsx` is the single source of truth for agent portrait rendering. It takes a `who` prop (agent ID string), looks up `AVATAR_CFG[who]`, and renders an SVG portrait at any size. It is already used correctly on Roster, TeachScreen, TestTeamScreen, and AssignWorkScreen.

Three locations opted out and built hand-rolled letter circles instead:
1. **DashboardScreen — task cards** (line ~81): `task.agent[0]` first letter in a `<div>` circle
2. **DashboardScreen — chat message headers** (line ~349, ~357): `agent.name[0]` first letter in a `<div>` circle
3. **StepList.jsx** (lines 24–37): a local `AgentInitials` component that derives initials from the agent name

Michelle Manning (PP-01) also has no entry in `AVATAR_CFG`, so any `<AgentAvatar who="michelle" />` call falls back to Chloe's portrait. This session adds her entry.

This session replaces all three hand-rolled circles with `<AgentAvatar>` and adds Michelle to `AVATAR_CFG`.

## 5. DESIGN RULES

- All tokens from `src/tokens.js` — `T.brass`, `T.card`, etc. Never hardcode hex.
- `AgentAvatar` signature: `<AgentAvatar who={agentId} size={number} ring={boolean} />`
  - `who`: agent ID string — must match a key in `AVATAR_CFG`; falls back to `chloe` if unknown
  - `size`: pixel dimension (width and height) — use 36 for task cards, 18 for chat/step inline
  - `ring`: `true` adds a dashed ring border; `false` for inline small uses
- Do not add `<AiBadge />` — avatars are not AI-generated content
- Do not add `<FeatureBadge>` wrapping the avatar — place it on the outer card wrapper instead

// test-avatar-01.mjs
// S-AVATAR-01 — RO-06 Avatar consistency sweep

let passed = 0;
let failed = 0;

function assert(label, condition) {
  if (condition) {

function resolveAvatar(who) {
  return AVATAR_CFG[who] || AVATAR_CFG.chloe;
}

assert("unknown agent ID falls back to chloe entry", resolveAvatar("unknown_agent") === AVATAR_CFG.chloe);
assert("null who falls back to chloe", resolveAvatar(null) === AVATAR_CFG.chloe);
assert("undefined who falls back to chloe", resolveAvatar(undefined) === AVATAR_CFG.chloe);
assert("known agent 'michelle' resolves correctly", resolveAvatar("michelle") === AVATAR_CFG.michelle);

console.log("\n── Test Suite: Chat message agentId shape ──");

// A: Chat message objects have agentId — simulate messages array
const mockMessages = [

// A: Task objects must have agentId (not just agent name string)
