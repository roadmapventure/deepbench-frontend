## 2. CONTEXT

BUG-12 was marked Done in S-CONTENT-01b but QA revealed AI Enrichment still shows "No calls yet" in the AI Audit panel. Root cause confirmed:

The BUG-12 log block in `api/prompt/ai-enrichment.js` (lines ~253–284) writes two fire-and-forget rows to `ai_activity_log` when reflect and synthesis run — one with `ai_type: 'reflect'` and one with `ai_type: 'synthesis'`. Both inserts are **missing `tenant_id: 'global'`**.

`hydrateFromSupabase()` in `useAIActivity.js` queries with `.eq('tenant_id', 'global')`. Any row without `tenant_id` set is excluded from the query result. The rows may be inserting (if the column is nullable) but are never returned to the UI.

The frontend `AI_TYPE_TO_SERVICE` mapping (`reflect → 'ai-enrichment'`, `synthesis → 'ai-enrichment'`) is correct — that was the S-CONTENT-01b fix. The only missing piece is `tenant_id` in the two server-side inserts.

Fix: add `tenant_id: 'global'` to both insert bodies.
