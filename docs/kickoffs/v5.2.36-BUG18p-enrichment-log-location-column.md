## 2. CONTEXT

Every reflect and synthesis fire-and-forget log insert in `ai-enrichment.js` has been failing silently since S-CONTENT-01a. Root cause: the inserts include `location: 'ai-enrichment'` but the `ai_activity_log` table has no `location` column. The actual column is `feature`. Supabase returns error 42703 (column does not exist), which is swallowed by `.catch(e => console.warn(...))`. Result: zero reflect or synthesis rows ever written to `ai_activity_log`, and Dan never appears in the AI Audit By Agent view.

**Confirmed schema** (via Supabase query 2026-06-24):
```
id, tenant_id, ai_type, feature, model, agent_id, task_id,
input_tokens, output_tokens, latency_ms, knowledge_tier,
cost_usd, created_at, patterns_used
```

`feature` (text) is the correct column. `location` does not exist.
