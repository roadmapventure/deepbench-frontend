## 2. CONTEXT

PE-11 adds the Edit Course inline sub-view to the Training tab in the Personnel File screen.

Currently the EDIT button on each training card is a disabled placeholder. This session wires it up.

**How it works:**
- Clicking EDIT on an active training card opens `AddCourseView` in edit mode. The left sidebar nav and page header remain fully visible — only the card list area is replaced, exactly like Add Courses already works.
- Edit mode pre-populates the form (title, category, jurisdiction, priority, triggers, teaching note) from the existing entry. No file upload. No re-vectorization. No call to `/api/ingest` or `/api/extract`.
- Save calls PATCH to `/api/knowledge-entry` with the updated metadata fields.
- NIGP parity decision: both EDIT and DELETE buttons are only shown for `agent.trainable && entry.status === "active"` — not shown for disabled entries, not shown for non-trainable agents.

**What is NOT changing:**
- No re-vectorization. The document's vector embeddings in RAG are not affected.
- The Add Courses flow is unchanged.
- No new screen or route. Everything stays within the Personnel File window.
