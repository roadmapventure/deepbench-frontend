## 2. CONTEXT

**Root cause:**

The `suggest-goal` fetch in `handleDeliverableSelect` has no client-side timeout. When the Vercel serverless function cold-starts (as seen in the 504 on run 1), the RAG query inside `suggest-goal` runs slowly or fails. The server catches the error and writes `data: [DONE]\n\n` immediately — with zero text chunks. The client receives `[DONE]`, `setGoalStreaming(false)` fires, but `accumulated` is `''` so `setGoal` was never called with content. The goal textarea stays blank with no feedback. The user sees an empty field and has no idea the suggestion failed.

**Failure sequence:**
1. Run 1 → 504 (Vercel cold start — function timed out before responding)
2. Run 2 → works (function now warm)
3. Run 3 → fresh page load → function cold again → suggest-goal returns [DONE] with no text → blank goal field, no error shown

**Fix (Option B):**
- Add 10-second `AbortController` timeout on the suggest-goal fetch
- On `[DONE]` with zero accumulated text, show a one-line inline notice below the goal textarea: `"Suggestion unavailable — describe your goal above."` The textarea remains editable; the notice disappears the moment the user starts typing.

- Notice text: `"Suggestion unavailable — describe your goal above."` — exact wording, no changes.
- Notice style: `mono`, `fontSize: 10`, `color: T.muted` — matches existing muted helper text in the screen.

- Notice does NOT appear during streaming (only after `[DONE]` with zero text).
- No toast, no modal, no red error color — this is a soft informational state, not a failure alert.

// test-bug16.mjs — BUG-16 goal suggestion timeout tests
// Tests: B (state logic), C (timeout + failure path)

let passed = 0;
let failed = 0;

  if (condition) { console.log(`  ✓ ${label}`); passed++; }
  else { console.error(`  ✗ FAIL: ${label}`); failed++; }

// ── B1: Timeout fires after 10s ───────────────────────────────────────────────
console.log('\nB1: AbortController timeout');
{
  let aborted = false;
  const controller = new AbortController();
  controller.signal.addEventListener('abort', () => { aborted = true; });
  const id = setTimeout(() => controller.abort(), 100); // use 100ms for test speed
  await new Promise(r => setTimeout(r, 150));
  assert('controller aborts after timeout', aborted);
  clearTimeout(id);
}

// ── B2: Zero-text [DONE] sets goalSuggestFailed ───────────────────────────────
console.log('\nB2: Zero-text [DONE] detection');
{
  let accumulated = '';

  onDone();

  const onDone = () => { if (!accumulated) goalSuggestFailed = true; };
  onDone();
