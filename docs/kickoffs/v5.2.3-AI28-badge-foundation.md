## 2. CONTEXT

DeepBench has an AiBadge component (✦ diamond chip) that renders on every AI-touched element in the app. The tooltip label currently uses freeform strings ("AI Chat", "AI Routing", "Plan Generation", etc.). This session begins a three-session sweep (S-AI-BADGE-01/02/03) to replace all labels with AI Pattern names from the AI Services catalog — the same pattern names shown in the AI Audit "By Pattern" section. The goal: a user hovers a badge anywhere in the app, sees pattern names, opens AI Audit, and sees the same terminology.

This session does three things:
1. Creates `src/aiPatterns.js` — a shared constants file with all pattern label strings and an agent→pattern map (used by S-AI-BADGE-02 for step card logic)
2. Updates `SharedUI.jsx` — adds a `built` prop to AiBadge for a greyed/dashed visual when a pattern exists in the catalog but is not yet implemented; removes the AiBadge from FlagCard's Recommended Action section (SVC-12 Procurement Flags is deterministic — no AI badge per architecture rules)
3. Updates `DashboardScreen.jsx` — replaces 3 existing freeform badge labels with pattern constants; adds a new AiBadge to the "Create a new Task" button (DB-22) showing all patterns invoked across the full Create New Task flow

**No api/ files are touched. No step card logic in this session (that is S-AI-BADGE-02).**
