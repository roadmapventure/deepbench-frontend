## Section 2 — CONTEXT

This is a UI polish session with no logic changes, no API calls, and no new features. It fixes five specific visual issues discovered during the S-BENCH-UX-01 design review:

1. RosterScreen stats strip is missing `<Corners>` and uses a non-standard border
2. RosterScreen "+ Add a Player" button is invisible (no border, no background, reads as a footnote)
3. RosterScreen "+ Add Training" agent card buttons are missing the AiBadge
4. PersonnelScreen PlaybookTab guardrail textareas use `T.paper` — a token that does not exist — leaving textareas with no background color
5. PersonnelScreen shows a redundant "← The Bench" back button in the top header (AppShell headerProps), alongside the centered Bench nav tab which already handles that navigation
6. PersonnelScreen left sidebar agent identity strip shows avatar + name + code but no active status chip

### Task 1 — RosterScreen: Fix stats strip (Corners + border + "Add a Player" button)

**File:** `src/screens/RosterScreen.jsx`

**What to change:** The navy stats strip (the `<div>` with `background:T.navy` that contains bench stats). Currently:
```jsx
<div style={{background:T.navy,padding:"10px 20px",marginBottom:20,display:"flex",alignItems:"center",gap:28,borderBottom:`3px solid ${T.brass}`}}>
```

Change to:
```jsx
<div style={{background:T.navy,padding:"10px 20px",marginBottom:20,display:"flex",alignItems:"center",gap:28,border:`1px solid rgba(182,135,58,.3)`,position:"relative"}}>

```

The `borderBottom: 3px solid ${T.brass}` is replaced with the style guide navy panel border. `position: "relative"` is required for Corners to render correctly. `<Corners color={T.brass}/>` goes as the first child inside the div.

**Also in Task 1 — upgrade "+ Add a Player" button:**

Current button (inside the stats strip, after the flex spacer):
```jsx
<button
  onClick={() => navigate("/bench/new")}
  style={{ marginLeft:"auto", fontFamily:body, fontSize:12, color:"#8fa3bf", background:"transparent", border:"none", cursor:"pointer", letterSpacing:.5 }}>
  + Add a Player
</button>
```

Replace with:
```jsx
<button
  onClick={() => navigate("/bench/new")}
  style={{ marginLeft:"auto", fontFamily:body, fontSize:12, color:T.brassLight, background:"transparent", border:`1px solid rgba(182,135,58,.4)`, padding:"5px 14px", cursor:"pointer", letterSpacing:.5 }}>

</button>
```

## Section 5 — DESIGN RULES

All tokens from `src/tokens.js`. Never hardcode hex values.

| Token | Use in this session |
|-------|---------------------|
| `T.navy` | Stats strip background, badge label color on brass buttons |
| `T.brass` | Corners color on navy panel, Add a Player border |
| `T.brassLight` | Add a Player text color |
| `T.moss` | ACTIVE chip color, moss-colored agent Add Training button |
| `T.muted` | Agent code label |
| `T.cardAlt` | Guardrail textarea background (replaces T.paper) |
| `T.lineSoft` | Sidebar identity strip border-bottom |
| `T.line` | Sidebar chip borders |

**AiBadge color rule (§4):** Badge color must match button label color. On brass background: `<AiBadge style={{ color: T.navy }} />`. On moss background: `<AiBadge style={{ color: "#fff" }} />`. Badge placed outside the button element, inside a flex wrapper.

**Corners usage:** `<Corners />` requires `position: "relative"` on the wrapping div. Use `<Corners color={T.brass} />` on navy panels.

**FeatureBadge:** `<FeatureBadge id="RO-02" />` in RosterScreen's AgentCard outer div (already has `position:"relative"`). Do not add a second RO-02 badge if one already exists.
