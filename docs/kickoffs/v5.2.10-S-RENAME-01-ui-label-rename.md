The platform is migrating from "Task" to "Work Order" as the primary unit of work. This session renames all user-visible labels only. Internal variable names (`taskId`, `task`, `tasks`), Supabase table names (`tasks`), API routes (`/work/:taskId`), and AI audit log location strings are explicitly out of scope — they remain unchanged. Future sessions will handle the backend rename.
