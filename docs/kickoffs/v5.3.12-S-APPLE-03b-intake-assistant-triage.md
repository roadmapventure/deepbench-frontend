## 2. CONTEXT

Sam Reyes (CI-05, The Intake Assistant) already has a static identity in `agents.js` (`AG-23`, shipped `S-APPLE-01a`) with a full 23-field entry, `AVATAR_CFG`, and `AGENT_PRONOUNS` — verified present, `STANDARDS.md` Section 11 already satisfied, no agent changes this session. She has no Data Room access (`data_room_access: []`, confirmed live) and needs none — her job is routing judgment over decisions and failures that are handed to her already-formed, not answering questions from source data.

**Design doc §5.8 (Commit Triage):**
> Trigger: director commits Forecast or Correct.
> Input: `{ intent: "forecast | correct", hypothesis, stress_test, correction_text }`
> Output: `{ route_to: ["reasoner", "data-expert"], data_expert_reason, disputed_chunk_id }`
> Always routes to Reasoner on Forecast/Correct. Routes to Data Expert only when the correction disputes a specific existing chunk's accuracy, not just an interpretation — a judgment call, not a fixed rule.

**Design doc §5.9 (Failure Triage):**
> Trigger: Proofreader hard-fails (`quality-gate`'s `status: 'failed'` response, `S-APPLE-02c`).
> Input: `{ guardrail_failure, original_question }`
> Output: `{ recommend_escalate, suggested_research_request }`
> Turns a dead-end failure into an offered next step (route to Data Expert's Escalate) rather than a wall.

**Behavior trait content (design doc §5b, already locked):** *"triage judgment — when a correction needs Data Expert's patch vs. Reasoner alone"* — Sam's general reasoning style across both calls: always route the mandatory next step, add the optional second destination only when the input genuinely warrants it, and never leave the reasoning for including it unstated.
