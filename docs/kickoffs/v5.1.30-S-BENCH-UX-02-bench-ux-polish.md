## Section 2 — CONTEXT

This session fixes 13 visual and logic issues found during the design UX review of the Roster, Resume, Training, Playbook, and Add Courses screens. No new features. No new API endpoints. No Supabase schema changes.

**Issues being fixed:**

**RO-08 / T-1 / AC-1 / AC-4 — AiBadge audit:**
- Roster card "Add Training" button: badge invisible on brass/moss background — needs navy-tinted chip treatment
- Training tab stats strip "+ Add Courses": badge is outside the button — move it inside (brass bg → navy chip treatment)
- Add Courses drop zone "Browse File": remove AiBadge entirely — file browse does not use AI
- Add Courses save CTA "Teach [Name] this document": missing AiBadge — ingest calls AI, badge required with navy chip treatment

**R-1 / P-1 — AddConfigForm hardcoded label:**
`AddConfigForm` in `ResumeTab.jsx` is shared between Resume (type="role_prompt") and Playbook (type="output_format"). The section header ("New Role Prompt") and save CTA ("Add Role Prompt") are hardcoded regardless of `type` prop. Playbook users see wrong labels.

**R-2 — handleSaved doesn't zero existing defaults (ResumeTab):**
When a new role prompt is saved with `is_default: true`, `handleSaved` appends without clearing other defaults. Playbook's `handleFormatAdded` has the correct pattern — sync Resume to match.

**R-3 — `T.paper` token undefined (ResumeTab):**
Four edit inputs use `background: T.paper`. There is no `T.paper` token in `tokens.js`. These inputs get `undefined` and fall back to browser white. Replace with `T.cardAlt`.

**R-4 — Skill Ladder card missing Corners (ResumeTab):**
The Vitals card above it has `<Corners />` and `position: "relative"`. The Skill Ladder card directly below it has neither. Inconsistent with every other card in the app.

**P-3 — handleSetDefault optimistic update (ResumeTab):**
ResumeTab's `handleSetDefault` optimistically maps `is_default` locally. PlaybookTab re-fetches from server after PATCH — that is the correct pattern. Server is source of truth. Sync Resume to refetch.

**AC-2 — "What Changes" + "Onboarding Checklist" cards missing Corners (PersonnelScreen):**
The "Projected Impact" card has `<Corners />` and `position: "relative"`. The two cards directly below it ("What Changes", "Onboarding Checklist") have neither. Inconsistent.

**AC-3 — Onboarding Checklist item 2 condition wrong (PersonnelScreen):**
"Priority & flags assigned" currently checks `addState === "ready" && addForm.title !== ""`. Priority and flag triggers are independent of the title field. Condition should match item 1: `addState === "ready" || addState === "saving"`.

**AC-5 — "Replace" button label color inconsistent (PersonnelScreen):**
The "✎ Replace" ghost button uses `color: T.brassDeep`. Every other ghost button in the file uses `color: T.mutedDeep`. Fix to match.

Locate `handleSaved` (around line 177). When a new config is saved with `is_default: true`, zero out existing defaults before prepending. Match the Playbook pattern:
```jsx
// Before
const handleSaved = (config) => {
  setConfigs(prev => [...prev, config]);
  setShowAdd(false);
};

// After
const handleSaved = (config) => {
  setConfigs(prev => {
    const updated = config.is_default
      ? prev.map(c => ({ ...c, is_default: false }))
      : prev;
    return [config, ...updated];
  });
  setShowAdd(false);
};
```

**P-3 — handleSetDefault refetch:**
Locate `handleSetDefault` (around line 148). Replace optimistic local update with server refetch:
```jsx
// Before
const handleSetDefault = async (id) => {

    await apiPatchConfig(id, { is_default: true });
    setConfigs(prev => prev.map(c => ({ ...c, is_default: c.id === id })));
    showToast("Default updated ✦");
  } catch (e) { showToast("Failed to set default", "⚠"); }

// After
const handleSetDefault = async (id) => {
  try {
    await apiPatchConfig(id, { is_default: true });
```

## Section 5 — DESIGN RULES

**Tokens (never hardcode hex):**
- `T.navy` = `#12243c`
- `T.brass` = `#b6873a`
- `T.brassDeep` = `#886224`
- `T.brassLight` = `#e4c786`
- `T.moss` = `#5a7538`
- `T.card` = `#f8f2e2`
- `T.cardAlt` = `#f2ead4`
- `T.mutedDeep` = `#58503a`
- `T.muted` = `#786d52`
- `T.line` = `#c8bb9a`
- `T.lineSoft` = `#d8cbac`
- `T.ink` = near-black form input color
- `T.flag` = `#a83319`

**AiBadge on brass/moss backgrounds (RO-08 rule — locked this session):**
Badge stays inside the button. Pass style overrides:
- On brass bg: `style={{ color: T.navy, background: "rgba(18,36,60,0.12)", border: "1px solid rgba(18,36,60,0.2)" }}`
- On moss bg: `style={{ color: "#fff", background: "rgba(255,255,255,0.18)", border: "1px solid rgba(255,255,255,0.3)" }}`
- Do NOT show AiBadge on cancel state of any toggle button
- Do NOT show AiBadge on file browse actions that do not call AI

**Corners:**
Every card that has a visible border must have `<Corners />` as first child and `position: "relative"` on the wrapper. No exceptions.

**Ghost button label color:** Always `T.mutedDeep`. Never `T.brassDeep`.

**Fonts:** `display` = Fraunces, `body` = Inter, `mono` = JetBrains Mono. All imported from `../../tokens.js` or `../tokens.js`.
