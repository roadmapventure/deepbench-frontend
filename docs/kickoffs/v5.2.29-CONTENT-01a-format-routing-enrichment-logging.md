## 2. CONTEXT

The Prompt Service pipeline has three steps: DB Assembly → AI Enrichment → Request & Receivable. Currently Michelle's capability (`project-manager`) includes the execution-plan Format Skill, which means the format schema (tool_use JSON contract) arrives in Column 2 of the Prompt Evolution Modal alongside Michelle's Identity, Intent, and Behavior. This violates the content/display separation principle.

**What changes:**
- The execution-plan Format Skill moves to Alex Reeves (ED-01, Screen Controls Editor), who owns the `screen-controls` capability.
- Supabase: two changes: (a) remove execution-plan from `capability_skill_profiles` for `project-manager`; (b) add `format_skill_profile_slug` + `display_agent_id` to every tile in SP-PM-03's `traits.deliverables` array so plan.js can resolve the format routing without hardcoding.
- `api/plan.js` — in `prompt-service` action: after `enrichPrompt()` returns, fetch Alex's Format Skill by slug, append `=== OUTPUT FORMAT ===` section to `enriched.system_prompt` as the final instruction, override `format_contract`, fetch Alex's agent card.
- `api/plan.js` — in `preview-prompt` action: append Alex's format section to stage4 so it appears as the last addition in Column 4 of the Prompt Evolution Modal.
- `api/prompt/ai-enrichment.js`: REFLECT and Synthesis each make Haiku LLM calls but write nothing to `ai_activity_log`. Add two fire-and-forget Supabase inserts — one for reflect, one for synthesis — attributed to agent `dan`, at the end of `enrichPrompt()`.

**Why format is appended last:** Dan's full enrichment (RAG, REFLECT, Synthesis) builds professional intelligence without a schema constraint. Alex's output contract is the final instruction: "here is how to structure what you just produced." This is the format-last architecture principle locked in the design session.

**Vercel function count:** 12 (unchanged — no new api/ files).

let displayAgentCard = null;
if (format_skill_profile_slug) {
  const supabaseUrl = process.env.SUPABASE_URL;
  const supabaseKey = process.env.SUPABASE_SERVICE_KEY;
  if (supabaseUrl && supabaseKey) {

        `${supabaseUrl}/rest/v1/skill_profiles?slug=eq.${encodeURIComponent(format_skill_profile_slug)}&select=*&limit=1`,

      if (fspRes.ok) {

          const outputType = traits.output_type || 'json';

            handler: traits.handler || 'store',

      if (display_agent_id) {

          `${supabaseUrl}/rest/v1/agents?id=eq.${encodeURIComponent(display_agent_id)}&select=name,role,specialty,bio&limit=1`,

        if (agRes.ok) {
