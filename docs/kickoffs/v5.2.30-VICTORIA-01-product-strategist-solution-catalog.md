## 2. CONTEXT

**Why this session exists:**

S-CONTENT-01a removed the execution-plan Format Skill from Michelle's `project-manager` capability. This broke `loadDeliverables()` in CreateWorkOrderScreen.jsx — it traverses Michelle's capability to find a Format Skill, reads `traits.deliverables`, and renders tiles. With the Format Skill gone, Step 2 renders empty and the user can't proceed.

**Root cause:** Tile loading and prompt assembly were coupled to the same DB row. This session decouples them permanently.

**The fix + feature:**

Victoria Chen (SP-01) — Head of Product Strategy — is a new named platform agent who owns a `solution-catalog` capability. Her skill profile's `traits.solutions` array IS the product menu. The tile loader reads from Victoria's skill profile directly by slug, filtered by `pm_agent_id` matching the selected PM. No more traversal through the PM's capability.

**Business logic:**
- Victoria defines what DeepBench offers clients
- Michelle plans the work
- Each solution entry declares its own routing: which PM handles it, which format skill, which display agent
- Add a new solution → update Victoria's traits in Supabase → appears on screen. No code change ever.
- `pm_agent_id` filter means different PMs can offer different solution sets in the future. Today all solutions route through Michelle.

**Three tile states:**
- `active: true` → live tile, clickable
- `active: false, coming_soon: true` → greyed tile, "Coming Soon" badge, not clickable, shown in roadmap section
- `active: false, coming_soon: false` → hidden completely

**Victoria's UI presence:**
After Michelle is selected (Step 1), Victoria's avatar chip appears above the tile grid with a consultation message. She speaks to the user on behalf of the firm. Two sections render:
1. "Here's what we can help you with right now:" → live tiles
2. "Available in our next roadmap:" → coming soon tiles (greyed)

**Vercel function count:** 12 (unchanged — no new api/ files).

```js
// FEATURE: AW-29 — load solutions from Victoria Chen's solution-catalog skill profile
// Filtered by pm_agent_id matching the selected PM agent.
// Three states: active:true = live | active:false+coming_soon:true = roadmap | active:false+coming_soon:false = hidden
const loadDeliverables = async (agent) => {
  try {
    const { data, error } = await supabase
      .from('skill_profiles')
      .select('traits')
      .eq('slug', 'solution-catalog')
      .single();

    if (error || !data?.traits?.solutions) return;

    const agentSolutions = data.traits.solutions.filter(
      s => s.pm_agent_id === agent.id
    );

    const live = agentSolutions.filter(s => s.active === true);
    const roadmap = agentSolutions.filter(s => s.active === false && s.coming_soon === true);

    setDeliverables(live);
    setRoadmapDeliverables(roadmap);
  } catch (e) {
    console.error('[loadDeliverables] error:', e);
  }
};
```
