## 2. CONTEXT

### What is broken

**Michelle вЂ” Task Planning:**  
`AssignWorkScreen.jsx` calls `logAICall({ type:"planning" ... })` in two places but passes no `agentId`. The By Activity Type section in AI Audit shows planning call counts, but Michelle never appears in the By Agent section. She is the planning agent and should own every planning call.

**Susan вЂ” Document Extraction:**  
`TeachScreen.jsx` and `PersonnelScreen.jsx` both call `logAICall({ type:"extraction" ... })` with no `agentId`. Susan is the Trainer Agent (TR-08). Document Extraction is her capability. She should appear in By Agent with counts for every extraction call.

**Susan вЂ” Knowledge Reinforcement (+ bug fix):**  
`PersonnelScreen.jsx` calls `logAICall({ type:"knowledge-reinforcement" ... })` after ingest. This is a bug вЂ” the correct key in `AI_TYPES` is `"reinforcement"` (not `"knowledge-reinforcement"`). Because the key doesn't exist in the catalog, these calls are orphaned вЂ” they never appear in By Activity Type aggregation. Fix the key AND attribute to Susan.

**`useAIActivity.js` descriptions вЂ” stale:**  
The `extraction` description references "Teach Agent screen" as the only location (Training tab also uses it). The `reinforcement` description says "Brent synthesizes run output" вЂ” Brent doesn't use this type; Susan does. Both need agent-neutral rewrites.

### What we are building (AI-18)

Wire `agentId` into every capability call that belongs to a named agent:

| Capability | Agent | Files |
|-----------|-------|-------|
| `planning` | `"michelle"` | `AssignWorkScreen.jsx` (2 call sites) |
| `extraction` | `"susan"` | `TeachScreen.jsx` (1), `PersonnelScreen.jsx` (1) |
| `reinforcement` | `"susan"` | `PersonnelScreen.jsx` (1) вЂ” also fix wrong key |

Update `useAIActivity.js` catalog descriptions to be agent-neutral.

### Expected result

After this session:
- **By Agent** section in AI Audit shows Michelle Manning В· PP-01 with planning call counts
- **By Agent** section shows Susan Smith В· TR-08 with extraction + reinforcement call counts
- **By Activity Type** вЂ” "Knowledge Reinforcement" row now accumulates counts (was broken due to wrong key)
- Brent's `summarization` rows are unchanged (different type, different call site)
